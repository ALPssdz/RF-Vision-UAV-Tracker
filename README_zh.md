## 1. 系统引言
RF-Vision-UAV-Tracker 是一套分布式的多模态无人机（UAV）探测与预警系统。本系统通过聚合软件无线电（SDR）技术与边缘计算光学视觉技术，彻底规避了传统基于单一传感器的探测方案所固有的物理局限（例如雷达的近场盲极与光学易受无线电静默欺骗的缺陷）。系统底层架构采用非对称的带外（Out-Of-Band, OOB）传感器融合机制，以在复杂电磁与视觉环境下实现高鲁棒性的目标截获与取证记录。

## 2. 系统硬件架构
系统的硬件拓扑基于千兆以太网局域网（LAN）构建，连接了三个高度解耦的物理计算节点：

*   **射频传感探测节点 (ZYNQ-7020 + AD9364)**
    担任主干全向探测阵列。该节点利用 AD9364 收发器高达 56 MHz 的大瞬时调谐带宽，配接垂直极化双频天线，对 2.4 GHz 与 5.8 GHz ISM 频段实行连贯频谱测绘及特征跳频截获。
*   **视觉光电传感节点 (Kendryte K230)**
    担任天顶防空端点补偿节点。挂载 1080P 高清光学传感器，依靠其内部集成的 KPU (计算知识处理器) 执行硬件级别的 YOLO 算子推理。其物理部署于主要探测面上方，用于弥补全向射频天线的“天顶极化零陷（Zenith Null）”盲区。
*   **主控调度大核 (RK3588 Platform)**
    充当全局事件总线与聚合处理枢纽。执行位于 Python 层面的路由控制逻辑，负责解算网络协议层面各高负载矩阵，同时协调多模态并发数据流于 PyQt5 上位机界面同步显示并完成数据库持久化存储操作。

## 3. 非对称融合体制设计
有别于常规布尔代数下僵硬的 AND 逻辑（需两路全通），本系统实施独立、异步的越位触发器机制，以最大化预警系统召回率：

1.  **射频特征级抽取 (第一类触发源)**
    系统通过提取无人机通信专有图传（连续波特征/跳频图谱），一旦特征比对成功，即可判定周边空域遭到入侵并独立触发本地局域网级别告警。
2.  **视觉信令带外遥测 (第二类触发源)**
    为消除传统流媒体解码带来的高延迟代价，K230 边缘节点实施了带外（OOB）通信协议体系。视频画面数据通过标准的高带宽 RTSP/TCP 链路传送供人类复核，而其目标锁定推理结论（边界框坐标，置信水平）则通过完全解耦的 UDP 低纳秒级信令通道单向靶向广播。一旦获取正确的 UDP JSON 光电锁闭信息，系统在射频静默等环境下依旧可实现零延迟阻截点亮。

## 4. 软件栈与模块组织
本工程遵循严苛的模型-视图-控制器（MVC）编程范式，库目录拓扑如下：

*   `system_hub.py`: 中央主控执行引擎入口与主干循环管线控制权持有者。
*   `rf_zynq/`: 对接 SDR 解析、特征洗牌数字信号处理（DSP）硬件抽象库。
*   `vision_k230/`: 完全解耦的轻量级并发网络适配端（RTSP+UDP 并发接听池）。
*   `ui_qt/`: 静态展现应用层（纯 View 组件层），严格遵守物理沙盒，禁止干涉任何推流获取与底层交互逻辑。
*   `database/`: SQLite3 扁平文件交互适配器，留存并标定每一份融合证据热散点复合图斑。

## 5. 部署与装配指南
所有脚本命令务必在项目总根目录下执行。

### 5.1 本机运行环境装配
```bash
pip install pyadi-iio PyQt5 opencv-python numpy torch ultralytics
```

### 5.2 系统拉起口令
```bash
python system_hub.py
```
> 指令激活后，底层物理端口和网口将优先获取句柄并建链，在握手完全校验完毕后，上位系统进程才会被正确映射到图传屏上。
