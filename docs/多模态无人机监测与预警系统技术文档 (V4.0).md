# 多模态无人机监测与预警系统技术文档 (V4.0)

## 项目概述

### 项目背景

针对低空无人机非法飞行及安全隐患，设计一套能够对2.4GHz、5.8GHz 等常见频段和无人机的视觉图像进行实时监测与预警的综合系统 。

### 项目目标

构建基于“端-边”协同的分布式边缘计算网络，通过射频特征识别与 YOLO 视觉识别的双重融合，实现高准确率的无人机发现、识别与日志存档 。

## 系统总体架构与拓扑

系统采用高度解耦的三个硬件节点，通过 **5 口千兆工业级交换机** 组成局部局域网 。

### 物理节点定义 (Physical Node Definition)

系统由三个高度解耦的硬件节点组成，通过**五口千兆工业级交换机**实现物理互联与数据汇聚：

- **射频感知节点 (Pluto SDR: ZYNQ7020 + AD9363)** 
  
  - **核心职责**：负责 2.4GHz 和 5.8GHz 频段的底层电磁环境扫描 。
  - **技术实现**：利用 **PL 端**（FPGA）进行高速信号采样、FFT 频谱分析与特征提取（如带宽、能量包络）；通过 **PS 端**（ARM）经由 **LwIP 协议栈**进行数据封包 。
  - **输出特性**：通过千兆网口，以 **UDP 协议** 向主控发送轻量化频谱特征数据报，确保高实时性与低传输延迟 。
- **视觉感知节点 (Kendryte K230 端)** 
  - **核心职责**：作为独立视觉 AI 终端，负责实时画面捕捉与目标检测 。
  - **技术实现**：在 KPU（神经网络加速器）上部署训练好的轻量化 **YOLO 模型**，在边缘端完成深度学习推理 。
  - **输出特性**：通过**千兆网络**同步输出双路数据：一路为极简的语义识别结果（包含类别、置信度及坐标框 [x,y,w,h]），另一路为经过**硬件编码压缩的实时视频流（具体编码格式及推流协议将根据 K230 SDK 的实际测试结果选型）**，供主控端拉取并进行可视化预警展示。
- **中央主控与数据平台 (RK3588 端)** 
  
  * **核心职责**：系统“大脑”与交互中枢，负责多路数据汇聚、逻辑融合判定、全景监控展示及数据持久化。
  
  - **硬件配置**：采用 **8GB RAM**（支持多用户并发开发）与 **128GB M.2 NVMe SSD**（解决数据库高频写入瓶颈）。
  - **软件环境**：运行完整版 Ubuntu 系统，部署 MySQL 数据库及**基于纯 C++ 与原生 Qt 框架（Qt5/Qt6）开发的高并发、高性能上位机监控软件**。

### 数据流向逻辑 (Data Flow Logic)

系统采用分布式计算架构，将重算力任务下放至端侧，主控仅负责逻辑集成，具体流向如下：

1. **视觉识别流**：
   - K230 传感器采集原始图像，本地 KPU 瞬间完成 YOLO 推理，并利用内部视频编码引擎对原始画面进行压缩。
   - 识别结果（目标置信度与坐标JSON包）与**实时高清视频流**通过网络被同步、实时地推送到 RK3588。
2. **射频感知流**：
   - ZYNQ 节点通过 AD9363 采集射频信号，FPGA 完成并行 FFT 运算提取频域特征 。
   - 特征包（包含中心频率、功率强度等）通过 **UDP 数据报** 发送至 RK3588 的指定监听端口 。
3. **融合与存储流**：
   - **汇聚**：RK3588 开启多线程服务，并发接收来自两个节点的异步数据流 。
   - **融合**：算法逻辑对双路数据进行**时间戳对齐**。当射频信号特征与视觉识别目标在时空维度重合时，判定为高置信度无人机目标 。
   - **预警与存盘**：触发上位机界面报警弹窗；同时由数据库模块将报警时间、信号特征及视觉证据图（截图）存入 **M.2 SSD**，形成可回溯的历史日志记录 。

## 核心子系统详细设计 

### 射频感知前端 (ZYNQ7020 + AD9363) 

该子系统利用 SDR（软件定义无线电）架构实现对低空电磁环境的实时监控。 

- **PL 端 (FPGA 高速逻辑处理)**：
  - **并行运算**：利用 FPGA 的硬件并行性，实现 1024/2048 点高速 FFT（快速傅里叶变换）运算，将时域信号实时转化为频域特征。 
  - **特征提取**：实时计算信号功率谱密度（PSD），识别突发能量脉冲，提取中心频率、信号带宽及平均功率等物理层特征。 
- **PS 端 (ARM 通信网关)**：
  - **低延迟传输**：基于高性能 LwIP 协议栈封装特征数据报，采用 **UDP 传输协议** 确保数据以亚毫秒级延迟推送至主控中心。 
  - **流量控制**：根据网络拥塞情况动态调整数据推送频率，确保特征提取结果的连续性。 
- **频段覆盖与策略**：精准监测 **2.4 GHz 与 5.8 GHz** 消费级无人机跳频频段，支持通过频率模板匹配技术过滤环境中的 Wi-Fi 与蓝牙干扰。 

### 视觉识别节点 (Kendryte K230) 

该节点作为独立的边缘 AI 计算单元，负责将非结构化的图像信息转化为结构化的语义结果。 

- **边缘 AI 推理 (KPU 加速)**：
  - **模型部署**：在 K230 的 KPU（神经网络加速器）上部署深度优化的轻量级 **YOLOv8n / YOLOv11n** 模型。 
  - **实时性能**：利用硬件加速实现高帧率（FPS）的目标检测与动态跟踪，有效应对无人机在复杂背景下的快速移动。 
- **图像预处理**：节点内部自动完成图像的缩放、色彩空间转换（RGB/YUV）及归一化，减少主控端的计算负担。 
- **视频与语义双轨输出**：
  - **结构化报文**：向主控高速发送包含“类别（Class）、置信度（Confidence）、边界框坐标（BBox）”的极简 JSON 报文，用于主控后端的融合判定。
  - **实时视频推流**：在端侧利用内置引擎完成**视频画面的压缩编码**，并开启网络推流服务。配合千兆交换机，在保证上位机获得流畅监控画面的同时，避免原始无损视频造成的网络拥堵。

### 中央主控中心 (RK3588) 

作为系统的“大脑”，负责全局调度、深度融合与人机交互。 

- **硬件规格与资源分配**：
  - **强劲算力**：采用 8 核 CPU（4x A76 + 4x A55）架构，配合 **8GB RAM**，支持多名组员通过远程 SSH 同时在线协作开发。 
  - **高速存储**：搭载 **128GB M.2 NVMe SSD**，为 Ubuntu 系统运行、MySQL 数据库高频 I/O 及报警图片存储提供高性能底座。 
- **多模态融合引擎**：
  - **并发处理**：运行多线程异步服务，一侧监听来自 ZYNQ 的千兆网口 UDP 特征包；另一侧通过网络接收 K230 的 JSON 识别结果，**并利用专门的解码线程（如 OpenCV 或专门的流媒体库）实时拉取并解析 K230 传回的视频流**。
  - **时空判定逻辑**：执行双模态逻辑交叉验证——当射频能量异常的时间戳与视觉识别目标的时空方位一致时，判定为“高置信度目标”，触发系统报警。 
- **UI 系统与交互**：
  - **实时监控**：在 Ubuntu 桌面端运行 **C++ Qt 图形化界面**。系统底层利用 C++ 的多线程调度与硬件加速机制，结合 **QCustomPlot 库**实现高频射频频谱瀑布图的无损极低延迟绘制；同时结合 **OpenCV C++ API**（或底层 FFmpeg）实时解码网络推流，转化为 `QImage`/`QPixmap` 对象，绝对流畅地同步播放带识别框的高清监控视频。
  - **报警与存档**：触发声光告警，并自动将报警瞬间的所有维度特征及截图存入 MySQL 数据库，供后续查询。

## 数据库与日志系统设计

本系统将数据库定位为全系统的“数据黑匣子”与“证据溯源中心”，不仅承载实时数据的持久化，更为后期的事故分析与性能优化提供底层支撑 。

### 存储架构与性能优化 (M.2 NVMe SSD)

- **高频写入支持**：针对无人机监测中射频数据更新频率高（秒级多次）的特点，利用 **M.2 NVMe SSD** 的高随机读写性能（IOPS），消除传统存储介质在多线程并发写入时的 I/O 瓶颈 。
- **读写分离策略**：上位机界面读取历史日志与后台数据写入流程互不干扰，确保在查看历史记录时，实时的监测预警逻辑不受影响 。

### 核心数据表结构定义

为了方便开发人员对接，定义以下核心数据模型：

- **检测记录表 (Detection_Records)**：用于记录系统运行期间的所有**射频特征**。

  | **字段名称 (Field Name)** | **数据类型 (Data Type)** | **约束 (Constraint)**       | **说明 (Description)**               |
  | ------------------------- | ------------------------ | --------------------------- | ------------------------------------ |
  | **Record_ID**             | BIGINT                   | PRIMARY KEY, AUTO_INCREMENT | 唯一记录 ID，设为主键                |
  | **Timestamp**             | DATETIME(3)              | NOT NULL                    | 检测发生的准确时间，支持毫秒级精度   |
  | **Center_Frequency**      | DOUBLE                   | NOT NULL                    | 信号中心频率（单位：MHz / GHz）      |
  | **Bandwidth**             | FLOAT                    | NOT NULL                    | 信号占用带宽（单位：MHz）            |
  | **RSSI**                  | FLOAT                    | NOT NULL                    | 接收信号强度指示（单位：dBm）        |
  | **SNR**                   | FLOAT                    | NOT NULL                    | 信噪比，用于评估信号质量（单位：dB） |

- **报警日志表 (Alarm_Logs)**：用于存储触发融合逻辑后的高置信度事件。

  | **字段名称 (Field Name)** | **数据类型 (Data Type)** | **约束 (Constraint)**       | **说明 (Description)**                             |
  | ------------------------- | ------------------------ | --------------------------- | -------------------------------------------------- |
  | **Alarm_ID**              | BIGINT                   | PRIMARY KEY, AUTO_INCREMENT | 唯一报警记录 ID，设为主键                          |
  | **Record_ID**             | BIGINT                   | FOREIGN KEY                 | 关联 `Detection_Records` 表的 ID，用于回溯射频特征 |
  | **Target_Class**          | VARCHAR(50)              | NOT NULL                    | 识别出的无人机型号或类别（如：DJI_Mavic_3）        |
  | **Confidence**            | FLOAT                    | NOT NULL                    | 视觉识别置信度（0.00 ~ 1.00）                      |
  | **BBox_Info**             | VARCHAR(100)             | NOT NULL                    | 目标在画面中的坐标框信息 [x, y, w, h]              |
  | **Evidence_Image_Path**   | VARCHAR(255)             | NOT NULL                    | 报警瞬间截图在 SSD 中的绝对存储路径                |

- **系统心跳表 (System_Status)**：记录 ZYNQ、K230 各节点的在线状态与主控板的负载情况，用于系统健康诊断 。

### “历史记录与取证”模块功能

上位机通过对上述表结构的调用，实现功能闭环：

- **可视化日志列表**：在 UI 界面实现响应式数据表格，动态展示报警历史 。
- **多维检索与导出**：支持按“时间区间”、“频段分布”或“置信度等级”进行组合筛选，并支持一键生成 PDF/CSV 格式的监测简报 。
- **证据回溯 (Forensics)**：
  - 用户点击任一日志条目，上位机自动调取 SSD 中对应的 **关键帧截图**。
  - 同步调取该时间点前后的射频特征参数，在界面上实现“音视图”三位一体的证据还原，确保监测结果具有司法/行政取证效力 。

##  项目开发计划与阶段性目标

本项目分为四个主要开发阶段。每个阶段均设定了明确的技术交付物与实现目标，以确保多节点协同开发的顺利推进。

### 第一阶段：需求分析与环境搭建 

**核心任务**：确定系统功能需求，制定分布式技术路线和系统架构，完成底层硬件的采购与基础环境配置 。 **阶段实现目标（交付物）**：

- **硬件就位**：RK3588 主控板（含 M.2 SSD）、K230 视觉开发板、ZYNQ7020+AD9363 射频系统全部到位并完成基础供电测试。
- **主控环境跑通**：RK3588 成功从 NVMe SSD 启动 Ubuntu 桌面系统，完成 Python 基础运行环境搭建。
- **数据库建表**：在 RK3588 上成功部署 MySQL，并完成“检测记录表”、“报警记录表”等核心数据表的创建测试。

### 第二阶段：核心模块独立开发 

**核心任务**：各物理节点（视觉、射频、主控）断开互联，进行独立的功能开发与算法验证 。 **阶段实现目标（交付物）**：

- **视觉节点 (K230) 目标**：成功在 K230 的 KPU 上运行 YOLO 模型。能够利用摄像头在本地画面中推理目标，通过网络稳定发送 `[类别, 坐标, 置信度]` 数据，**并调研跑通适用于 K230 的视频压缩与网络推流方案，确保外部设备能拉取到流畅的监控画面**。
- **射频节点 (ZYNQ) 目标**：FPGA 端成功实现对 2.4G/5.8G 频段的 FFT 频谱分析 ；ARM 端成功通过千兆网口使用 UDP 协议向外连续发送打包好的频域特征数据。
- **上位机 (RK3588) 目标**：完成前端 UI 界面原型的编写，能够显示静态的图表框架和视频占位符，并能通过代码成功向 MySQL 插入模拟测试数据。

### 第三阶段：边缘协同与系统集成 

**核心任务**：打破节点孤岛，实现 K230、ZYNQ 与 RK3588 之间的通信与数据交互，完成上位机系统的数据贯通 。 **阶段实现目标（交付物）**：

- **通信链路打通**：RK3588 后台能够同时、无阻塞地接收 ZYNQ 发来的 UDP 射频数据和 K230 发来的视觉结果。
- **多模态融合生效**：融合代码逻辑验证成功——当代码同时检测到特征匹配的射频信号和视觉置信度大于阈值的目标时，系统能自动生成一条“高置信度报警”并写入数据库。
- **可视化界面点亮**：打通 **Qt 信号与槽（Signals and Slots）机制**，实现数据接收底层线程与 UI 渲染主线程的彻底解耦。上位机能利用 **C++ 与 QCustomPlot** 实现高帧率无卡顿渲染射频频谱图，并同步解码播放视频流。

### 第四阶段：系统实测、优化与进阶 

**核心任务**：将系统移至户外真实环境，对系统进行整体测试，优化边缘计算性能并提高系统稳定性 。 **阶段实现目标（交付物）**：

- **核心指标达标**：在户外实测中（针对大疆等常见消费级无人机），实现一定距离内的稳定报警，漏报率与误报率降低至项目预期范围内。
- **系统并发优化**：长时间运行（如连续运行24小时）后，RK3588 系统内存无泄漏，MySQL 写入不卡顿，千兆网口数据无严重丢包。
- **进阶探索（射频指纹）**：如果在基础功率/带宽检测上已达标，尝试在 RK3588 的融合判定逻辑中引入初步的射频指纹（RF Fingerprinting）比对机制，利用信号瞬态特征尝试区分无人机的具体型号。
